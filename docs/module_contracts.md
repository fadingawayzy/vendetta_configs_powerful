# Module Contracts

## run.py
Задача:
- запустить приложение
- мониторить ресурсы

Не должен:
- содержать бизнес-логику pipeline

## app/__init__.py
Задача:
- собрать приложение и запустить сервисы

Не должен:
- содержать детали fetch/filter/scan/export

## app/tasks/pipeline/fetch.py
Задача:
- получить контент из внешних источников
- извлечь сырые ссылки
- сохранить сырые кандидаты

Не должен:
- оценивать качество соединения
- экспортировать результат пользователю

## app/tasks/pipeline/filter.py
Задача:
- отобрать допустимых кандидатов

Не должен:
- сканировать TCP / SSL
- публиковать результаты

## app/tasks/pipeline/scan.py
Задача:
- проверить доступность конфигов
- вычислить ping / tier
- подготовить итоговые данные

Не должен:
- получать сырой контент источников
- выдавать HTTP response

## app/tasks/pipeline/export.py
Задача:
- выгружать уже проверенные конфиги

Не должен:
- повторно сканировать сеть
- фильтровать сырые данные

## app/utils/parser.py
Задача:
- разобрать ссылку и извлечь поля

Не должен:
- ходить в БД
- отдавать HTTP response

## app/utils/net_tools.py
Задача:
- сетевые проверки

Не должен:
- управлять Telegram-ботом
- менять БД напрямую без необходимости

## app/utils/web_server.py
Задача:
- отдавать HTTP endpoints
- возвращать подписки и singbox-конфиги
- вести понятные диагностические логи

Не должен:
- молча проглатывать ошибки
- дублировать маршруты
- запускать pipeline самостоятельно

## database/methods.py
Задача:
- операции с БД

Не должен:
- содержать Telegram/UI/web-логику
