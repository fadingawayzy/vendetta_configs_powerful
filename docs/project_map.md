# Project Map

## Проект
Telegram-бот и веб-сервис для агрегации и выдачи VLESS-конфигов из открытых источников.

## Главная цель
1. Получить открытые источники с конфигами.
2. Извлечь сырые VLESS-кандидаты.
3. Отфильтровать кандидатов.
4. Проверить доступность и качество.
5. Сохранить в БД.
6. Выдать пользователю лучшие конфиги через Telegram и HTTP endpoints.

## Главные модули

### run.py
Точка входа. Запускает приложение и монитор ресурсов.

### app/__init__.py
Оркестратор старта:
- подключает роутеры
- поднимает web server
- запускает scheduler
- запускает фоновое обновление подписок
- запускает polling бота

### app/tasks/pipeline/fetch.py
Собирает сырые ссылки и сохраняет кандидатов.

### app/tasks/pipeline/filter.py
Фильтрует кандидатов по правилам проекта.

### app/tasks/pipeline/scan.py
Проверяет доступность, ping, tier и качество конфигов.

### app/tasks/pipeline/export.py
Экспортирует лучшие конфиги наружу.

### app/utils/parser.py
Парсит ссылки и извлекает поля конфига.

### app/utils/net_tools.py
Сетевые проверки TCP / SSL / ping.

### app/utils/web_server.py
HTTP endpoints для выдачи подписок.

### database/methods.py
Операции чтения и записи в БД.

## Поток данных
Source -> Fetch -> Filter -> Scan -> Database -> Export / Telegram / HTTP

## Принцип работы с GPT
GPT должен менять только один модуль за раз и не трогать весь проект целиком.
